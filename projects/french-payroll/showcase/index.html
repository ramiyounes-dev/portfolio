Comment ça marche

Ce système simule un traitement de paie batch en COBOL avec une couche de présentation .NET / Blazor. Toute la logique de calcul est reproduite fidèlement côté client pour cette démonstration.

Architecture réelle

En production, le flux est :

  • GNU COBOL lit les fichiers employés et variables mensuelles
  • Calcule le brut, toutes les cotisations (salariales et patronales), le PAS, le net
  • Produit des fichiers plats en COMP-3 (décimal condensé)
  • ASP.NET Core API parse les fichiers COBOL et sert du JSON
  • Blazor Server affiche le tableau de bord

Structure du bulletin de paie

Conforme à l'arrêté du 25 février 2016, chaque bulletin suit cet ordre :

  • Identification employeur / employé (SIRET, NAF, matricule, NTT)
  • Période de paie et date de paiement
  • Salaire de base (taux horaire × heures, ou forfait mensuel)
  • Éléments variables (heures sup à 25% et 50%, primes, absences)
  • Cotisations salariales détaillées (maladie, vieillesse, CSG, retraite AGIRC-ARRCO...)
  • Net imposable, prélèvement à la source (barème neutre 2024)
  • Net à payer

Pourquoi COBOL ?

L'arithmétique COMP-3 (décimal condensé) garantit une précision au centime sans dérive de virgule flottante — essentiel quand on calcule des millions d'euros de cotisations. Le traitement batch correspond naturellement au cycle mensuel de la paie.
